# 登录问题 FAQ

## Q1: 无法通过 SSH 登录跳转节点

**问题描述**: 使用 ssh 命令连接跳转节点时失败，提示连接超时或拒绝连接。

**可能原因**:
- 公钥未配置或配置错误
- 动态码输入错误或过期
- SSH 配置文件有误（known_hosts 未更新）
- 跳转节点服务未启动

**解决方案**:
1. 检查本地 `~/.ssh/id_rsa.pub` 公钥是否已添加到目标节点的 `~/.ssh/authorized_keys`
2. 确认动态码在有效期内（动态码通常30秒有效）
3. 清除旧的 known_hosts：`ssh-keygen -R <hostname>`
4. 检查 SSH 服务状态：`systemctl status sshd`

---

## Q2: 公钥登录失败，提示 "Permission denied"

**问题描述**: 使用公钥登录时持续提示权限拒绝。

**可能原因**:
- 公钥内容格式不正确（多了或少了一些字符）
- `authorized_keys` 文件权限不正确（应为 600）
- `.ssh` 目录权限不正确（应为 700）
- 用户主目录权限不正确

**解决方案**:
1. 检查公钥格式，应以 `ssh-rsa AAAA...` 开头
2. 修正权限：
   ```bash
   chmod 700 ~/.ssh
   chmod 600 ~/.ssh/authorized_keys
   chmod 755 ~
   ```
3. 确认公钥已正确添加到目标机器的 `authorized_keys` 文件

## Q3: 动态码获取失败或显示不正确

**问题描述**: 无法获取登录动态码，或动态码显示与预期不符。

**可能原因**:
- 动态令牌应用未同步时间
- 动态令牌设备更换未重新绑定
- 动态码生成器应用故障

**解决方案**:
1. 检查手机端动态令牌应用时间是否正确
2. 尝试重新同步动态令牌应用时间
3. 如更换设备，需联系管理员重新绑定动态令牌
4. 备选方案：使用短信动态码或其他备用认证方式

---

## Q4: SSH 连接经常断开

**问题描述**: SSH 连接建立后经常无原因断开。

**可能原因**:
- 网络不稳定
- SSH 客户端设置未启用保活机制
- 防火墙或安全策略导致连接超时

**解决方案**:
1. 在 SSH 客户端配置文件中启用保活：
   ```
   Host *
       ServerAliveInterval 60
       ServerAliveCountMax 3
   ```
2. 检查本地网络连接稳定性
3. 联系网络管理员确认防火墙设置
